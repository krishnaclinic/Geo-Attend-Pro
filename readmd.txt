v0.2.3 discussion of requirements:
Good discussion point. Let me break down the current
     design flaw and the options.

     Current Problem:

     - Today's attendance is loaded per-day (where('date',
       '==', today))
     - If a staff punches IN at 11pm and forgets OUT, the
       next day is a clean slate → system asks for IN again,
        leaving the previous shift dangling
     - No cross-day shift tracking exists

     Best approaches I see:

     Option A — Shift-Locked (closest to what you want)

     - Introduce a status field on the employee document:
       lastShift: { inTimestamp, date, autoClosed }
     - On page load, check if there's an open shift (IN
       without OUT across ALL days, not just today)
     - If open shift exists → force punch OUT before
       allowing a new IN
     - If open shift is > 24hrs → auto-close it as "Absent (
       incomplete)" and allow fresh IN
     - This binds the staff to finish their running shift

     Option B — Auto-Midnight Close

     - A cron/cloud function (or client-side check) auto-
       closes any open shift at midnight
     - Each day is independent
     - Simpler but doesn't enforce shift completion

     Option C — Hybrid (recommended)

     - Shift-locked model (Option A) for the current day
     - Auto-close stale shifts > 24hrs with "Absent" status
     - Visual reminders within the app (browser
       notifications if tab is open)
     - This gives you the enforcement you want + handles
       edge cases

     For notifications: since this is a static web app (no
     service worker/scheduled tasks), we can:

     - Show in-app alerts/reminders when the page is open
     - Use the Browser Notification API for push-style
       alerts
     - But cannot guarantee delivery if the tab is closed —
       that would need a backend (Firebase Functions + FCM)

     What's your preference — do you want the shift-locked
     model (must close open shift first), and for
     notifications, are in-app reminders while the tab is
     open sufficient, or do you need actual push
     notifications?
